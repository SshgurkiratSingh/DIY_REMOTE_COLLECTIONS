Receiver_v1.1.2_esp_now

Legacy receiver variant that validates two-way reply behavior and telemetry packet flow.

Telemetry bridge role

This revision bridges one-way listeners and direct hardware receivers by returning structured reply payloads.

Reply packet model

struct_reply {
  uint8_t sensorValue;
  uint8_t counter;
  uint8_t flags;
};

When to use

Use for validating RX callbacks on transmitter before moving to full motor-control receiver deployment.

Build / Flash

cd Receiver_v1.1.2_esp_now
pio run
pio run --target upload
