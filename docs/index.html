ESP32 controller stack

Handheld ESP-NOW remote firmware with OLED UI and receiver telemetry.

This repo bundles transmitter and receiver firmware variants for a DIY ESP32 remote, including persistent target storage, a captive portal for receiver registration, and a compact serial-debug receiver.

Overview

What’s in the project

Transmitter

Reads joystick, potentiometer, toggles, push buttons, and a rotary encoder, then transmits a packed control frame over ESP-NOW.

Receiver

Accepts incoming control data, handles dual motor control directly via mapped pins (e.g. for BTS7960 drivers), prints it over serial, and returns a small reply payload for link validation.

On-device UI

Uses a 128x64 SSD1306 OLED for HUD, telemetry screens, target selection, settings, and device info pages.

Setup

Build and pair the devices

1. Flash the receiver

Open ESP32_Receiver-Direct-v2.0 in PlatformIO and upload it.

cd ESP32_Receiver-Direct-v2.0
pio run
pio run --target upload
pio device monitor

Record the MAC address printed in the serial monitor.

2. Flash the transmitter

Open ESP_NOW_V1.2_OLED and upload the main handheld firmware.

cd ESP_NOW_V1.2_OLED
pio run
pio run --target upload

In settings, choose Add Tgt(AP), connect to ESP-NOW-REMOTE, then visit 192.168.4.1.

SSID: ESP-NOW-REMOTE
Password: 12345678
Portal URL: http://192.168.4.1
Max saved targets: 5

Control payload

  uint16_t joyX;
  uint16_t joyY;
  uint16_t potValue;
  bool toggle1;
  bool toggle2;
