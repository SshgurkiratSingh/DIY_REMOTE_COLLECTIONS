ESP32 controller stack

Handheld ESP-NOW remote firmware with OLED UI and receiver telemetry.

This repo bundles transmitter and receiver firmware variants for a DIY ESP32 remote, including persistent target storage, a captive portal for receiver registration, and a compact serial-debug receiver.

Overview

What’s in the project

Transmitter

Reads joystick, potentiometer, toggles, push buttons, and a rotary encoder, then transmits a packed control frame over ESP-NOW.

Receiver

Accepts incoming control data, prints it over serial, and returns a small reply payload for link validation and telemetry experiments.

On-device UI

Uses a 128x64 SSD1306 OLED for HUD, telemetry screens, target selection, settings, and device info pages.

Setup

Build and pair the devices

1. Flash the receiver

Open Receiver_v1.1.2_esp_now in PlatformIO and upload it.

cd Receiver_v1.1.2_esp_now
pio run
pio run --target upload
pio device monitor

Record the MAC address printed in the serial monitor.

2. Flash the transmitter

Open ESP_NOW_V1.1_OLED and upload the main handheld firmware.

cd ESP_NOW_V1.1_OLED
pio run
pio run --target upload

In settings, choose Add Tgt(AP), connect to ESP-NOW-REMOTE, then visit 192.168.4.1.

SSID: ESP-NOW-REMOTE
Password: 12345678
Portal URL: http://192.168.4.1
Max saved targets: 5

Packets

Data structures on the link

Control payload

struct_message {
  uint16_t joyX;
  uint16_t joyY;
  uint16_t potValue;
  bool toggle1;
  bool toggle2;
  bool push1;
  bool push2;
}

Reply payload

struct_reply {
  uint8_t sensorValue;
  uint8_t counter;
  uint8_t flags;
}

Notes

Current limitations

Implemented later

The settings UI shows a channel entry, but channel editing is not implemented in the current transmitter firmware.

Naming scope

The repository name mentions LoRa and BLE, while the current checked-in code focuses on ESP-NOW transmitter and receiver firmware.
